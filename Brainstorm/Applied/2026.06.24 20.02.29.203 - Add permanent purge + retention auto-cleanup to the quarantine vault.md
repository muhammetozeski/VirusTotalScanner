# Add permanent purge + retention auto-cleanup to the quarantine vault

Kategori: feature | Efor: M | Rank: 6

## Ne
QuarantineVault.cs exposes only Quarantine/List/Restore — there is no way to permanently delete a held .VIRUS file (confirmed: no Purge anywhere). Add QuarantineVault.Purge(entry) (delete the .VIRUS file, drop its manifest entry, Save) and PurgeOlderThan(int days). In QuarantineVaultDialog.cs (today Restore/Close only) add a 'Kalıcı sil' button behind a ConfirmGates gate and an 'Eski kayıtları temizle' action, and show the vault's total reclaimable size (sum of VaultFile lengths) in the dialog footer. Optional RetentionDays setting auto-purges on startup.

## Neden
Confirmed malware currently piles up in the vault forever and the user can never finish the job, reclaim the disk, or be certain it is gone. Restore-only is half a remediation flow; the destructive final step is the biggest missing piece of the quarantine lifecycle, and the manifest+id scheme already makes Purge a few lines.
