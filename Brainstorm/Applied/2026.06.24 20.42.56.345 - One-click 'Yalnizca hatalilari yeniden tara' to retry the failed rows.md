# One-click 'Yalnizca hatalilari yeniden tara' to retry the failed rows

Kategori: ux | Efor: S | Rank: 3

## Ne
Add a 'Yalnizca hatalilari yeniden tara' action-bar button (plus context-menu item / F-key) in ScanQueueControl that collects every row with BucketOf(i)==Bucket.Error (Status==Failed) and re-runs exactly those paths via StartScan(paths, recurse:false). Today failed rows are only parked in PendingOutbox (ScanScheduler.cs:248,265) with no in-grid retry. Reuse the existing Error bucket and the Shift+J/K error-walk so the retry set matches what the user is already navigating.

## Neden
Big sweeps routinely leave a scatter of Failed rows from transient 429s and network blips; recovering them now means hand-multiselecting each red 'Hata' row and Ctrl+R. A dedicated retry-failed action turns post-sweep cleanup into one click exactly when a real user needs it.
