# Pending-lookup outbox (offline self-heal)

Cost: M

A pending.json outbox recording every file whose VT lookup couldn't complete because VT was unreachable (no network after Polly exhausts retries, all keys exhausted, or keyless GUI timed out / captcha declined). Stores md5+sha256, path, size, TrustResult, failure reason — NOT a fake verdict. The scan row shows 'VT'ye ulasilamadi — sirada' instead of generic Hata. A background reconciler drains it on a connectivity signal (NetworkChange.NetworkAvailabilityChanged or the scheduler heartbeat): replay each stored hash through cache->GUI->API, on success write the real report to cache and remove from outbox. Deduped by sha256, survives restarts, clearable from UI. No re-hash/re-walk (replays hashes) — distinct from crash-resume and Downloads-watch.

_From round-3 autonomous brainstorm (10 proposer-vs-critic groups, 2026.06.24 04.2x)._
