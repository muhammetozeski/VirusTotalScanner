# Scan inside archives the download-watcher catches, not just the container hash

Kategori: protection | Efor: M | Rank: 5

## Ne
In DownloadsWatcher.HandleAsync (lines 71-86), when a stabilized file is archive-class (ExecExts already lists .zip/.7z/.rar/.iso at line 13), call ArchiveExpander.IsExpandable/ExpandToTemp to extract to a temp folder and run each member through the same trust-prefilter + cache + keyless-GUI pipeline HandleAsync already uses on a single file (refactor the line 71-86 verdict logic into a per-file helper, loop over members, clean the temp dir in finally). Today line 76 hashes the container blob as one chunk — its hash never matches a VT report, so the file silently increments Cleared. A malicious member raises the existing ThreatFound naming the inner file ('invoice.zip > setup.exe').

## Neden
Most download-borne malware arrives zipped. The watcher already ACCEPTS .zip/.7z/.rar/.iso but clears any packed payload as safe — an active false sense of security on the download guard. ArchiveExpander already exists and is wired into the manual scanner (ScanScheduler.ExpandArchives), so reusing it here closes the most common infection vector with almost no new code.
