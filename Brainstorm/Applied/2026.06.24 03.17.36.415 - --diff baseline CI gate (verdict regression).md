# --diff baseline CI gate (verdict regression)

CLI --diff <baseline.json> compares the current scan vs a prior --report JSON, keyed by sha256. Classify each file NEW / REGRESSED (detection count rose or clean->flagged) / UNCHANGED. Add --fail-on-new and --fail-on-regression exit gates on top of existing exit-code logic. Print a delta section + add a 'delta' block to json/text/html report. Current side served from cache (zero quota). The missing CI primitive: pass on steady state, trip only when something gets worse. Cost S.
