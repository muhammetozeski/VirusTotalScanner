# Mark major-engine and stale-signature rows in the engine grid

Kategori: ux | Efor: M | Rank: 5

## Ne
ScanDetailControl.Engines_CellFormatting (line 220) only reddens rows where r.IsDetection. Both separating signals are already public: MajorEngines.IsMajor(EngineName) and VtEngineResult.UpdatedUtc. Add a leading star / bold engine name on rows where IsMajor is true, and a grey tint + clock glyph on DETECTING rows whose UpdatedUtc is older than the same cutoff VtReport.StaleDetections already uses. Add a checkbox beside _showAll (ScanDetailControl.cs:25,61) - 'Yalnizca buyuk motorlar' - that switches the DataSource to major engines, mirroring the existing _showAll.CheckedChanged->Populate() pattern (line 309).

## Neden
The engine table is a flat wall of ~70 vendors where a real Microsoft/Kaspersky hit looks identical to an obscure false positive, and a hit on month-old signatures looks identical to a fresh one. Letting the user instantly see which detections come from engines that matter, and which are stale, is the single biggest 'is this real or noise' clarifier, using data already computed.
