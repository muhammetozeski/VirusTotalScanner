# Hands-off USB auto-scan option (skip the click-the-toast step)

Kategori: protection | Efor: S | Rank: 3

## Ne
OnRemovableInserted (MainForm.cs:51) only sets ToastAction.ScanUsb and shows a 6s 'click to scan' balloon; the scan starts only when the user clicks it. Add a Settings flag (AutoScanUsb, next to WatchUsb) with a SettingsControl checkbox that, when on, calls _scan.StartScan([drive], recurse:true, background:true) immediately — reusing the existing background auto-quarantine path in OnThreatFound(item, background:true) (MainForm.cs:332-344) — and shows only a result toast.

## Neden
Removable media is a top infection vector and the current flow still needs a deliberate click on a balloon that is easy to miss. StartScan already supports background:true and that path already auto-quarantines high-detection finds, so a truly passive auto-scan closes the window where an autorun payload sits unscanned because nobody clicked in time. Small change on shipped plumbing.
