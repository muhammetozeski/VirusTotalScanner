# Persist clean→threat flips as a standing 'Sonradan tehdit oldu' ledger

Kategori: protection | Efor: M | Rank: 7

## Ne
Add EscalationStore.cs (escalations.json, mirroring AllowlistStore's atomic-write pattern — confirmed it does not exist) and call it from HistoryReverdictService.CheckAsync where each escalation is added to the result list, so every confirmed clean→threat flip is recorded permanently with old/new ratio, SHA-256, first-scan date and flip date. Dedupe by hash; keep the first observed flip date. Surface a pinned 'Sonradan tehdit oldu' section above the grid in ScanHistoryControl.cs and a red count line on the overview (ScanOverviewControl.Refresh2, near the watch/quarantine counters). Today these escalations are shown once in HistoryReverdictDialog and discarded when it closes.

## Neden
'A file I cleared later turned malicious' is the highest-value security signal this app produces, and right now it evaporates the instant the dialog closes. A persistent ledger lets the user see WHICH past-trusted files went bad and when, even if they did not act during that one dialog session — turning a transient computation into a durable, actionable record.
