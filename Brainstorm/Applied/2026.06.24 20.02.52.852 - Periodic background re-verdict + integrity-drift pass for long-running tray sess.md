# Periodic background re-verdict + integrity-drift pass for long-running tray sessions

Kategori: protection | Efor: M | Rank: 9

## Ne
The watch-list re-query runs exactly once: StartWatchCheck (MainForm.cs:478) fires WatchService.CheckAllAsync after an 8s startup delay and never again; BaselineStore.VerifyAsync (path-keyed tamper detection) has zero references in MainForm and runs only from the manual Verify button. Add one user-configurable System.Windows.Forms.Timer (e.g. 6-12h) that, guarded by !Scheduler.IsRunning, re-runs WatchService.CheckAllAsync (keyless, zero quota), RecheckService.DueForRecheck+RunAsync against AppServices.Cache, and BaselineStore.VerifyAsync — raising the existing ShowWatchEscalations toast on any detection climb and an urgent toast for any DriftResult.IsAlarm. Route toasts through the existing NotificationGate.

## Neden
The app is built to live in the tray for days (start-hidden --tray, MinimizeToTray), but a borderline 0/72 file that turns malicious next week, or a pinned system binary silently swapped for malware, is never re-checked unless the user restarts and clicks. One recurring timer makes the zero-quota re-verdict and the already-built tamper detection genuinely passive instead of launch-only, with no new services.
