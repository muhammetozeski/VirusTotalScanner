# Publisher continuity check (signer-swap alert)

Persist per cached file the local Authenticode publisher CN (from TrustService.ExtractSigner during trust pre-filter) + version-resource identity (ProductName+OriginalFilename via FileVersionInfo) as two new HashCacheEntry fields. On a brand-new hash, scan cache for prior entries sharing the same normalized product identity; if earlier versions were consistently signed by X and the new file is now unsigned or signed by Y, raise a graded 'signer changed for this product' alert in the detail panel (now-unsigned=loud, different-publisher=warning, same-publisher bump=silent). Product-identity-keyed (not path-keyed like the integrity baseline); fires on first sight at any path. Zero quota.
