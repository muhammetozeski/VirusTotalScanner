# Reversible Quarantine Vault with Restore

Cost: M

Turn quarantine from one-way into reversible. On quarantine write a per-item JSON manifest in the quarantine folder: original full path, extension, sha256/md5, verdict+count at quarantine time, timestamp, Zone.Identifier origin; vault file uses a collision-safe id name. A 'Karantina kasasi' dialog lists held items with Restore: move back to the exact original path, restore extension/attributes/MotW, gated by ConfirmGate. Before restoring, RecheckService re-looks-up the current verdict (keyless): if cleared since quarantine (FP case) restore proceeds; if still flagged, warn loudly. Wires quarantine into the recheck/drift infra.
