# Named scan profiles (switchable setting snapshots)

Kategori: ux | Efor: M | Rank: 7

## Ne
Add a ScanProfileStore (JSON list, modeled on FolderSuppressionStore.cs: atomic write via AtomicFile.WriteAllText + BackupCorrupt on load failure) holding named snapshots of only the scan-relevant Settings keys: SkipSafeExtensionsOnScan, RiskWeightedOrdering, MaxConcurrentScans/MaxConcurrentUploads, HashCacheDays/ThreatCacheDays/RecheckPeriodDays, MaxFileSizeMB, TrustSkipSigned/TrustMicrosoftOnly. Reuse the per-setting ISettingSetup.Serialize/LoadFromStr plumbing that SettingsManager.ExportSettings/ImportSettings already use to capture and apply just those keys, then call SaveSettings(). A profile only overwrites its own subset, never the whole config. Surface it as a small dropdown on SettingsControl.BuildScanCard (save current / apply / delete named profile) plus 'Profil uygula: <ad>' entries injected into ScanQueueControl.Commands() so Ctrl+K switches profiles in one keystroke.

## Neden
Bir guc-kullanici ayni kutucuklari isten ise elle acip kapatiyor: 'derin adli tarama' (trust bypass, kucuk cache, genis eszamanlilik) ile 'hizli indirilenler kontrolu' (safe-ext atla, buyuk cache) arasinda. Tek global config + tek export/import bu gunluk gecisi karsilamiyor. Isimli profiller surtunmeyi kaldirir; alt yapi (per-setting Serialize/LoadFromStr, FolderSuppressionStore deseni, komut paleti) zaten hazir, sadece kucuk bir store + UI baglantisi eksik.
