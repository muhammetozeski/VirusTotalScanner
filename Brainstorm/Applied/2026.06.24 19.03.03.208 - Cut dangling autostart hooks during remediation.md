# Cut dangling autostart hooks during remediation

Kategori: protection | Efor: M | Rank: 7

## Ne
PersistenceHunter.cs (Find, line 16) is listing-only — Run/RunOnce values, Startup .lnk files, scheduled tasks — and HuntPersistence() in ScanQueueControl.cs (line 892) prints a manual note telling the user to fix it in regedit. Add PersistenceHunter.Remove(Hook): per hook Location, RegistryKey.DeleteValue for Run/RunOnce, File.Delete (moved into the vault folder first) for the Startup .lnk, 'schtasks /delete /tn <name> /f' for a task. Back up each removal reversibly first (write reg value name+data to a restore file in QuarantineFolder, move the .lnk into the vault). Expose as a per-hook 'Bu kancayi kaldir' button in the HuntPersistence dialog behind ConfirmGates, and offer it automatically right after a quarantine succeeds.

## Neden
After a file is quarantined the autostart entry pointing at it survives, so at next boot Windows keeps trying to launch the now-missing or restored binary. Today the only guidance is 'go edit the registry yourself'. Closing the persistence loop is the gap between 'flagged' and 'actually removed', and the backup makes it safely reversible.
