# Auto-surface persistence hooks the moment a file is quarantined

Kategori: protection | Efor: M | Rank: 8

## Ne
TryQuarantineUnlocking / QuarantineItem in ScanQueueControl.cs (lines 717-758) move the file but never call PersistenceHunter.Find — the Run-key/Startup/Task hunt runs only from the separate manual HuntPersistence menu item (line 922). After a successful quarantine, call PersistenceHunter.Find(originalPath) for that file and, if any hooks match, open the existing PersistenceHooksDialog so the dangling autostart can be cut in the same breath via PersistenceHunter.Remove.

## Neden
Quarantining the binary while leaving its Run key intact means Windows tries to launch a now-missing .VIRUS every boot — a visible broken-startup error and an incomplete cleanup. The hunt and the reversible Remove already exist; wiring them into the quarantine moment (when the path is fresh) closes a loop users nearly always forget to close by hand.
