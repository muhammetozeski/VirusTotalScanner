# 'Korumam ne kadar acik?' protection-coverage card on the overview

Kategori: protection | Efor: M | Rank: 3

## Ne
Add a coverage card to ScanOverviewControl (new row between the status banner and the drop-zone) reading the passive-protection toggles directly, each shown on/off with one-tap enable: Indirilenler izleniyor (Settings.WatchDownloads, ships FALSE per Settings.cs:99), USB otomatik tarama (Settings.WatchUsb), Zamanli tarama (SweepScheduler.IsInstalled()), Sag-tus menusu (ContextMenuInstaller.Verify()). Off rows render amber; the WatchDownloads row flips the setting and raises an event so MainForm calls StartDownloadsWatchIfEnabled() (MainForm.cs:280); the others jump to the relevant Settings section via the existing GoToTab event.

## Neden
WatchDownloads ships OFF and SweepScheduler is opt-in, so a user can run the app for weeks believing they are covered while nothing is actually watching new files. The shipped red/amber/green banner only answers 'is a known threat on disk now' — never whether anything guards the machine going forward. This card makes passive protection visible and one click to turn on, closing the biggest real-world gap.
