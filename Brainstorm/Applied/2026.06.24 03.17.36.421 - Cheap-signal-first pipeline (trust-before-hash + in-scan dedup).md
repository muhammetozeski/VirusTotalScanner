# Cheap-signal-first pipeline (trust-before-hash + in-scan dedup)

Two coupled perf changes in ScanScheduler.ProcessAsync, gated by a setting: (1) When SkipTrusted, run TrustService.Evaluate (works off the file handle, no hash) BEFORE the full MD5+SHA256 read, so trust-skipped signed files are never read end-to-end (huge on signed install trees). Known-good-list check stays after hashing. (2) In-scan content dedup: a per-run ConcurrentDictionary<md5, Lazy<Task<report>>> so the FIRST item with a given md5 does the lookup and every duplicate awaits+reuses it (closes the gap where _cache.Put only happens at item end, so parallel duplicates each hit VT). Optional run summary '142 files, 38 unique, 71 trust-skipped before hashing'. Reduces disk I/O, GUI scrapes, and quota at once.

_From round-2 autonomous brainstorm (10 proposer-vs-critic groups, 2026.06.24 03.1x)._
