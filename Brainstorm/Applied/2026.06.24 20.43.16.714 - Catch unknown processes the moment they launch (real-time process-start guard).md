# Catch unknown processes the moment they launch (real-time process-start guard)

Kategori: protection | Efor: M | Rank: 7

## Ne
Add ProcessStartGuard.cs subscribing to the WMI Win32_ProcessStartTrace event via ManagementEventWatcher (adds the System.Management NuGet package). For each new process pull its on-disk image path and run it through the same cheap-signal pipeline DownloadsWatcher.VerdictOne uses: TrustService.Evaluate/ShouldSkip (clears the signed majority), then HashCache.TryGet, then keyless GuiScrapeService only for a genuine unknown; a malicious hit raises the same ThreatFound event the watcher uses (wired at MainForm.cs:168 into auto-quarantine + toast). Gate behind a new Settings.WatchProcessLaunches flag, off by default, with per-path _inflight dedupe; degrade gracefully (log + disable) when not elevated since the trace needs admin (AdminHelper exists).

## Neden
RunningProcesses.ImagePaths is a one-shot snapshot fired only from a manual button (ScanQueueControl.cs:1304). A user who double-clicks a fresh unknown exe from chat/email that never touched the watched Downloads folder gets zero passive coverage until it is already running. Catching it at launch is the biggest passive-protection gap and reuses the entire existing verdict+quarantine path; steady state is zero-quota because the trust pre-filter and cache clear the known majority.
