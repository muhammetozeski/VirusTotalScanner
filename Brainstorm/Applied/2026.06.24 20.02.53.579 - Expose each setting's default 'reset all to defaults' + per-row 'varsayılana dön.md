# Expose each setting's default: 'reset all to defaults' + per-row 'varsayılana dön', plus guarded Settings export/import

Kategori: ux | Efor: M | Rank: 12

## Ne
Setting<T> already stores a never-shown 'readonly T DefaultValue' (Setting.cs:46). Add 'object DefaultValue' and 'bool IsDefault' to ISetting. In SettingsControl.BuildAboutCard() (line 554) add a 'Tüm ayarları varsayılana döndür' button that, after a confirm, resets every registered setting to its DefaultValue and saves; for simple toggle/numeric rows, append a subtle '• değişti — varsayılana dön' link beside any control whose live value differs from default that resets just that field in place. In the same card add 'Ayarları dışa aktar' / 'Ayarları içe aktar' that copy/parse the key=value lines of ConfigPathResolver.ConfigPath via Save/OpenFileDialog, deliberately omitting the DPAPI-bound EncryptedKeyVault line so the config is portable and safe to share.

## Neden
Users today cannot tell what they changed from the shipped defaults, cannot undo a bad tweak without hand-editing the .config, and cannot replicate a tuned setup (trusted publishers, watch folders, thresholds) across machines except by copying a file whose path most never see. Making 'what did I change' visible and one-click reversible removes the fear of experimenting, and guarded export/import makes the rich config portable — both reuse data the model already carries.
