# Path integrity baseline and drift alert

A path-keyed baseline alongside the content-keyed cache: when a file is clean/TrustedSkipped, 'pin to integrity watch' records path -> {sha256, size, signer, mtime}. A 'verify watched files' action (GUI button + CLI flag) re-hashes those paths locally and reports drift: unchanged / changed-now-clean / changed-now-flagged / missing. Alert loudly only when a previously-clean trusted path lost signer continuity (now unsigned or different publisher); a normal signed update is a quiet info note. VT consulted only when a watched hash changed and is unknown.

_From the autonomous brainstorm (10 groups, 2026.06.23 23.2x). Deduped from 5 near-identical 'find all copies' proposals + 5 distinct ideas._
