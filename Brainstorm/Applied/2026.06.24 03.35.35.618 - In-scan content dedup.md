# In-scan content de-duplication

The trust-before-hash reorder is done. Add the second half of the cheap-signal pipeline: a per-run ConcurrentDictionary<md5, Lazy<Task<VtFileReport?>>> in ScanScheduler so the FIRST item with a given md5 performs the VT/GUI lookup and every other item with the same md5 awaits + reuses that report, instead of each hitting VirusTotal independently. Closes the gap where _cache.Put only happens at item end, so parallel duplicates (node_modules, bundled runtimes, repeated installers) currently all miss the cache and each do their own lookup. Each row keeps its own path/verdict; they just share one lookup. Clear the map per run; the persistent cache is unchanged. Optional run summary 'N files, M unique hashes'.
