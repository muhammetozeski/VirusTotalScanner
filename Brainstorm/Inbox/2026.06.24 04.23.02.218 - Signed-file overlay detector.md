# Signed-file overlay detector

Cost: M

A BCL-only PE structural check (System.Reflection.PortableExecutable.PEReader) detecting bytes appended AFTER the Authenticode signature blob — the region the signature does not cover. Read the Certificate Table (Security Directory) offset+size; signed file-end = certTableOffset+certTableSize; any non-zero bytes beyond = an overlay the signature didn't authenticate. Show in the detail pane: 'Signed by X, but N bytes appended after the signature — NOT covered (could be a stuffed/trojanized signed binary)'. PAYLOAD: gate TrustService.ShouldSkip on it — a validly-signed file WITH a post-signature overlay is NOT auto-skipped; force the normal VT/keyless lookup + behaviours. Closes the evasion where malware is appended to a legit signed binary (WinVerifyTrust still returns Trusted -> currently skipped). FP risk (NSIS/SFX append data) handled as a signal not a verdict.
