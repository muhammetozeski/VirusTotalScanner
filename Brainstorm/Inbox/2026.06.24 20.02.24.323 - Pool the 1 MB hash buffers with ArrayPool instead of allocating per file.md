# Pool the 1 MB hash buffers with ArrayPool instead of allocating per file

Kategori: perf | Efor: S | Rank: 4

## Ne
All three hashing methods in HashService.cs allocate 'byte[] buffer = new byte[1 << 20]' per file: ComputeAsync (line 14), ComputeMd5Async (line 34), VerifyExpectedAsync (line 58). Each 1 MB buffer lands on the Large Object Heap, and with localDegree = ProcessorCount concurrent hashers (ScanScheduler.cs:121) a big folder is a steady stream of LOH allocations and Gen2/LOH GC. Rent from ArrayPool<byte>.Shared and return in a finally so each worker reuses one buffer; the read loop is unchanged. (Confirmed: no ArrayPool anywhere in the codebase today.)

## Neden
Removes constant LOH allocation during the hottest part of a large scan, cutting the GC pauses that show up as periodic UI hitches and lower files/sec. Pure win, no behavior change, a few lines per method.
