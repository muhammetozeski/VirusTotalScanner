# Hash-verify the vault file before restore (anti-tamper restore guard)

Kategori: protection | Efor: S | Rank: 9

## Ne
Add a QuarantineEntry.VaultSha set inside QuarantineVault.Quarantine() by hashing the file right after File.Move into the vault (works even when the VT Sha256 is null, e.g. copies quarantined via QuarantinePath). In QuarantineVault.Restore (QuarantineVault.cs:99), before the File.Move back, re-hash the on-disk .VIRUS and compare to VaultSha; if they differ return an error and have QuarantineVaultDialog.RestoreSelectedAsync show 'kasa dosyasi degismis - geri yukleme guvenli degil' and steer the user to Purge instead.

## Neden
Restore currently File.Move's the held .VIRUS back to its original trusted path with zero integrity check. If malware or another process swapped or edited the file while it sat in the vault, the user re-arms a tampered binary at a path the system trusts. Capturing the hash at quarantine time closes the one unguarded hole in an otherwise reversible flow, and covers the copy-finder case where Sha256 is null.
