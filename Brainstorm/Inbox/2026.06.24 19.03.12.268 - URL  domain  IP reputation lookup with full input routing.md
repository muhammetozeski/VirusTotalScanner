# URL / domain / IP reputation lookup with full input routing

Kategori: feature | Efor: L | Rank: 12

## Ne
Add keyless GUI-scrape + API-fallback lookups for the three non-file VT object types. In GuiScrapeService.cs, generalize the existing capture trick (today it only navigates to /gui/file/<hash> and captures /ui/files/..., line 54) to also navigate to /gui/url|domain|ip-address and capture the page's own /ui/urls|domains|ip_addresses response; in VtApiClient.cs add GetUrl/Domain/IpReportAsync. New VtUrl/Domain/IpReport model types. Surface everywhere input flows: a 'Baglanti/site ara' action-bar + command-palette entry auto-detecting URL vs IPv4 vs domain; extend ScanClipboard (ScanQueueControl.cs:667) so a copied http(s) URL or bare domain/IP routes here instead of the current 'bir dosya yolu, klasor ya da hash degil' dead end (line 676); add a 'Panodaki baglantiyi tara' tray item beside the existing 'Panodaki yolu tara'.

## Neden
The app answers 'is this FILE safe?' but has zero way to answer 'is this LINK / site / IP safe?' — the most common safety question a normal user has (a link in an email, a download mirror, a sketchy IP in a firewall log). VT's free GUI and API both serve these object types and the rotation/GUI-scrape/quota plumbing is already built and directly reusable. Routing it through the shipped clipboard and palette entry points means the user's most natural gesture (copy a suspicious link) finally works.
