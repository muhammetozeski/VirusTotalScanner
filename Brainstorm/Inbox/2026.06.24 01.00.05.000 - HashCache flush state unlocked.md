# HashCache flush state unlocked

Rank: 05 (round-3 kalite/bug brainstorm, 24->23->10, satir-numarali dogrulanmis)

BUG (S). HashCache.Put lock-free ConcurrentDictionary'yi mutasyona ugratip _dirty=true + MaybeSave'i _saveLock olmadan cagiriyor. MaybeSave _dirty/_lastSaveUtc okur, Flush _dirty re-check eder ama yalniz File.WriteAllText _saveLock icinde; _dirty(bool)/_lastSaveUtc(DateTime) coklu worker thread'den senkronsuz okunup yaziliyor. Iki es zamanli Put throttle gate'i gecebilir, bir thread'in _dirty=false'u digerince kacirilip save dusurulebilir. Fix: _dirty/_lastSaveUtc read-modify-write'i _saveLock icine al, kilit altinda re-check+reset.
