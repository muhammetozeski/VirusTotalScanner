# Keyboard 'select all visible threats' to feed batch quarantine

Kategori: ux | Efor: S | Rank: 4

## Ne
Add a shortcut (e.g. Ctrl+Shift+J) in ProcessCmdKey next to the J/K handlers (ScanQueueControl.cs:378-381) that sets row.Selected on every grid row whose DataBoundItem passes IsThreatish (ScanQueueControl.cs:1212) within the current filter/view, scrolls the first into view, and writes '12 tehdit secildi - Ctrl+Q ile karantina' into _summary. This bridges the existing J/K threat-walk and the existing QuarantineSelected batch path (which already reads _grid.SelectedRows).

## Neden
The app already invests in keyboard threat navigation (J/K) and keyboard batch quarantine, but there is no keyboard way to grab the entire threat set, so you still mouse-multiselect. Select-all-threats closes that loop so someone triaging an infected PC can clear a flagged batch entirely hands-on-keyboard in two keystrokes.
