# Full sandbox behaviours panel

The report-level behavior summary (tags + threat label) is done. Add the FULL sandbox detail by capturing the /ui/files/<hash>/behaviours (or behaviour_summary) response in the WebView2 GUI engine the same way the file report is captured: network contacts (domains/IPs), dropped/written files, registry keys + persistence, processes started, and MITRE ATT&CK techniques. Show as a collapsible 'what this file did when run' panel in the detail pane, especially valuable for 0-detection files. Parse defensively; many files have no behaviour report.
