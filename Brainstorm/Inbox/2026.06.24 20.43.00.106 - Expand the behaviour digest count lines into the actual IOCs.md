# Expand the behaviour digest count lines into the actual IOCs

Kategori: ux | Efor: M | Rank: 6

## Ne
GuiScrapeService.FetchBehaviourAsync (lines 189-194) fully populates VtBehaviour.Network/FilesWritten/Registry/Processes with real hostnames, IPs, dropped-file paths and registry keys, but BehaviourDigestBuilder.Build collapses each to a bare count ('N ag adresine baglaniyor') and discards the detail strings. Extend DigestLine (BehaviourDigest.cs) with an optional IReadOnlyList<string>? Details, populate it in Build, and make each count line in ScanDetailControl.RenderBehaviour expandable to reveal the top ~8 items per bucket, each with a Kopyala button and a 'domaini ara' link for network rows. Persist Details through BehaviourDigestCache (older cached digests have null Details and fall back to count-only).

## Neden
A non-expert deciding whether to trust a flagged file needs to SEE that it phones home to a sketchy domain or drops an exe into a startup folder; the bare count is a teaser, not evidence. The data is already fetched and thrown away, so surfacing it turns the digest into something the user can act on or paste to someone who can, at near-zero data cost.
