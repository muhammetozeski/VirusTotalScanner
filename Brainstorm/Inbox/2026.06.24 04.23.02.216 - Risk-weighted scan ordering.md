# Risk-weighted scan ordering

Cost: M

Before the parallel scan loop, compute a cheap local suspicion score per file and OrderByDescending so the scariest files get a verdict first under the 4/min keyless ceiling. In ScanScheduler.RunAsync after Expand, before building items, sort by: Mark-of-the-Web (Zone.Identifier ZoneId 3/4) strong, unsigned/untrusted (TrustService) higher, location (Downloads/%TEMP%/AppData Temp/browser cache) higher, freshness (created/modified last few days) higher, extension class (.scr/.js/.vbs/.bat/.cmd/.ps1/.hta/.lnk + double-extension invoice.pdf.exe) higher, cached-clean to the bottom. Computed on the existing off-UI Task.Run. A small 'risk' column/tooltip; gated by a setting (default on). Only the MotW/freshness/location/extension scoring is new; trust+cache reused.

_From round-3 autonomous brainstorm (10 proposer-vs-critic groups, 2026.06.24 04.2x)._
