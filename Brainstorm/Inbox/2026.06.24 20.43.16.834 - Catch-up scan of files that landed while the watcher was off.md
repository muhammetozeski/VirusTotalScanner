# Catch-up scan of files that landed while the watcher was off

Kategori: protection | Efor: M | Rank: 8

## Ne
Add a public CatchUpAsync(folders) to DownloadsWatcher that enumerates each watched folder for exec-class files (the existing ExecExts set, DownloadsWatcher.cs:12-13) whose LastWriteTimeUtc is newer than a stored Settings.LastWatchScanUtc, and feeds any not already covered by HashCache through the existing VerdictOne path. Call it from StartDownloadsWatchIfEnabled (MainForm.cs:324) right after Start(), then update LastWatchScanUtc. Bound the work to the mtime window so it is cheap.

## Neden
FileSystemWatcher only fires for files created while it is alive, so a download finished overnight (app closed) or any file that arrived before WatchDownloads was toggled on is silently never verdicted, yet the watcher's own doc-comment (DownloadsWatcher.cs:6) promises files are scanned 'the moment they land.' The mtime-windowed catch-up closes that gap with zero new UI and full pipeline reuse.
