# Folder-scoped suppression for build-output / dev directories

Kategori: feature | Efor: M | Rank: 10

## Ne
Add a path-prefix suppression list (folder-allowlist.json, mirroring AllowlistStore's atomic-write/Changed pattern) and a Settings card 'Klasor bazli sessizlestirme'. ScanScheduler checks it next to the hash-keyed AllowlistStore.Contains call (ScanScheduler.cs:201) and TrustSkips files whose full path is under a registered folder, reason 'Gelistirme klasoru (kullanici onayi)'. Seed the add-flow from the queue/overview context menu ('Bu klasoru sessizlestir').

## Neden
AllowlistStore is hash-keyed (keys on Sha256/Md5), so a developer's freshly-built exe gets a brand-new hash every compile and is re-flagged endlessly — the hash list structurally cannot cover it. A folder scope (e.g. \source\repos\MyApp\bin) is the only mechanism that suppresses an entire stream of ever-changing-hash but trusted output, the dominant false-positive case for anyone who builds software.
